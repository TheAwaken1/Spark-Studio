Recovery
Something stuck or broken? Every action below says exactly what it does — none of them delete downloaded models.

      

        

          
 Clear finished runs

          
Removes exited runs from the run list. Run history stays in the database; nothing running is touched.

          
        

        

          
 Remove orphan containers

          
Removes leftover spark-vllm-* / sparkrun_* containers that no live run controls. Containers owned by running workloads, active sparkrun jobs, and anything you manage yourself (like vllm_node) are never touched.

          
        

        

          
 Reset registry cache

          
Deletes the local mirror of the community recipe repos and forged YAML cache, then re-downloads fresh copies in the background. Fixes stale or corrupted recipe listings.

          
        

        

          
 Reset app database

          
Deletes saved recipes, run history, benchmark and eval history. Does not stop a running model and does not delete downloaded model files. There is no undo.

          
        

        

          
 Copy bug report

          
Copies a markdown report — system health, active run, recipe (secrets redacted), and the last 300 log lines — ready to paste into a GitHub issue or hand to an agent.

          
        

        

          
 Full reset (terminal)

          
If the app itself won't start, run these from the install folder:

          
rm -f data/spark_studio.db     # settings/history only
rm -rf env && ./start.sh       # rebuild Python env
./start.sh --doctor            # health report
